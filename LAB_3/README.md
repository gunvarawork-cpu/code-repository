## ภารกิจที่ 1: การออกแบบ Training Strategy
### 1.1.	การกำหนดประเภทของข้อมูล (Land Cover Classes)
ในการศึกษานี้ เลือกพื้นที่ศึกษาเป็นบริเวณจังหวัดอยุธยา ได้กำหนดประเภทการใช้ประโยชน์ที่ดินจำนวน 5 ประเภท (Classes) ได้แก่
1)	Urban (พื้นที่เมือง/สิ่งปลูกสร้าง) ครอบคลุมพื้นที่ที่มีสิ่งปลูกสร้าง เช่น อาคาร ถนน และโครงสร้างพื้นฐาน
2)	Water (แหล่งน้ำ) ครอบคลุมพื้นที่ที่เป็นน้ำ เช่น แม่น้ำ คลอง บึง และแหล่งน้ำธรรมชาติ
3)	Agriculture (พื้นที่เกษตรกรรม) ครอบคลุมพื้นที่เพาะปลูก เช่น นาข้าว ไร่ และพืชสวน
4)	Vegetation (พื้นที่พืชพรรณ) ครอบคลุมพื้นที่ป่าไม้และพืชพรรณหนาแน่น เช่น ป่าไม้หรือ vegetation ที่มีความหนาแน่นสูง
5)	Barren (พื้นที่โล่ง/ดินเปล่า) ครอบคลุมพื้นที่ที่ไม่มีพืชพรรณ เช่น ดินเปล่า หรือพื้นที่ก่อสร้าง
### 1.2.	วิธีการจัดหาข้อมูลตัวอย่าง (Training Samples)
ข้อมูลตัวอย่างถูกสร้างขึ้นโดยวิธี การวาด Polygon ด้วยตนเองใน Google Earth Engine (GEE) โดยอาศัยการตีความจากภาพถ่ายดาวเทียม Sentinel-2 ร่วมกับการแสดงผลในรูปแบบต่าง ๆ เช่น RGB, NDVI และ NDWI แนวทางนี้ช่วยให้สามารถกำหนด label ได้อย่างเหมาะสมกับลักษณะพื้นที่จริง และลดข้อผิดพลาดที่อาจเกิดจากการใช้ข้อมูลอ้างอิงสำเร็จรูปที่ไม่ตรงกับบริบทของพื้นที่ศึกษา
### 1.3.	การแบ่งชุดข้อมูล (Train/Test Split)
ในการทดลองนี้ ใช้วิธี การสุ่มแบ่งข้อมูลแบบ 80/20 (Random Split) โดยแบ่งเป็น
-	ชุดฝึกสอน (Training set): 80%
-	ชุดทดสอบ (Testing set): 20%
วิธี Random Split เป็นวิธีที่ง่ายและนิยมใช้ในงานทั่วไป แต่อาจมีข้อจำกัดด้าน Spatial Autocorrelation ซึ่งอาจทำให้ค่าความแม่นยำสูงเกินจริงเมื่อข้อมูลมีความใกล้เคียงกันในเชิงพื้นที่ เมื่อเปรียบเทียบกับ Spatial Cross-validation พบว่า วิธีหลังมีความแม่นยำเชิงพื้นที่สูงกว่า แต่มีความซับซ้อนในการดำเนินการมากกว่า
### 1.4.	การเลือก Feature
Feature ที่ใช้ในการจำแนกประกอบด้วย
#### 1)	Spectral Bands:
-	B2 (Blue)
-	B3 (Green)
-	B4 (Red)
-	B8 (Near Infrared: NIR)
-	B11 (Shortwave Infrared: SWIR)
#### 2)	Spectral Indices:
-	NDVI (Normalized Difference Vegetation Index) ใช้แยกพืชพรรณ
-	NDWI (Normalized Difference Water Index) ใช้แยกแหล่งน้ำ
-	NDBI (Normalized Difference Built-up Index) ใช้แยกพื้นที่เมือง
การเลือก Feature ดังกล่าวมีวัตถุประสงค์เพื่อให้ครอบคลุมลักษณะทางกายภาพของพื้นผิวแต่ละประเภท และช่วยเพิ่มประสิทธิภาพของโมเดลในการจำแนกข้อมูล
## ภารกิจที่ 2: การเปรียบเทียบอัลกอริทึม
ในการศึกษานี้ ได้ทดลองใช้อัลกอริทึมทั้งหมด 3 รูปแบบ ได้แก่
### 2.1 Random Forest (50 Trees)
-	Overall Accuracy: 0.7938
-	Kappa Coefficient: 0.7311
-	F1-score: [0, 0.93, 1.00, 0.83, 0.40, 0.25]
### 2.2 Random Forest (100 Trees)
-	Overall Accuracy: 0.7835
-	Kappa Coefficient: 0.7181
-	F1-score: [0, 0.93, 1.00, 0.81, 0.38, 0.25]
### 2.3 Gradient Tree Boost (GTB)
- Overall Accuracy: 0.7938
- Kappa Coefficient: 0.7327
- F1-score: [0, 0.87, 1.00, 0.82, 0.52, 0.33]
### 2.4 การวิเคราะห์ผล
จากผลการทดลองพบว่า 
-	Random Forest ที่ใช้ 50 ต้นไม้ (trees) ให้ผลลัพธ์ที่ดีกว่าแบบ 100 trees เล็กน้อย Random Forest (50 trees) และ Gradient Tree Boost ให้ Accuracy ใกล้เคียงกัน
-	Gradient Tree Boost ให้ค่า Kappa สูงที่สุด แสดงถึงความสอดคล้องของการจำแนกที่ดี
-	การเพิ่มจำนวน trees ใน Random Forest ไม่ได้ช่วยเพิ่มประสิทธิภาพในกรณีนี้ อาจเนื่องจากข้อมูลมีขนาดจำกัดและมีความซ้ำซ้อนเชิงพื้นที่
### 2.5 Metrics ที่ใช้ในการประเมิน
-	Overall Accuracy
-	Kappa Coefficient
-	F1-score (ราย Class)
-	Producer’s Accuracy
-	User’s Accuracy
การใช้หลาย metrics ช่วยให้สามารถประเมินประสิทธิภาพของโมเดลได้รอบด้านมากขึ้น
## ภารกิจที่ 3: การวิเคราะห์ Feature Importance
### 3.1 Feature ที่มีความสำคัญสูง
จากผลการวิเคราะห์ พบว่า
-	B11 (SWIR) มีความสำคัญสูงที่สุด
-	รองลงมา ได้แก่ NDWI, B2 และ B4
ผลลัพธ์ดังกล่าวสอดคล้องกับหลักการทางกายภาพ เนื่องจาก SWIR สามารถแยกความแตกต่างของความชื้นในดินและวัสดุพื้นผิวได้ดี ขณะที่ NDWI มีความไวต่อแหล่งน้ำ
### 3.2 การทดสอบ Feature Removal
เมื่อทำการตัด NDVI ออกจาก Feature Accuracy เพิ่มขึ้นจาก 0.7835 เป็น 0.8026 แสดงให้เห็นว่า NDVI อาจมีความซ้ำซ้อนกับข้อมูลจาก spectral band อื่น และไม่ได้เพิ่มประสิทธิภาพของโมเดลอย่างมีนัยสำคัญในกรณีนี้
## ภารกิจที่ 4: การประเมินความไม่แน่นอน
### 4.1 Class ที่มีความสับสนสูง
จาก Confusion Matrix พบว่า
-	Urban และ Barren มีความสับสนกันสูง
-	Agriculture และ Forest มีความคล้ายคลึงกัน
สาเหตุเกิดจากลักษณะ spectral ที่ใกล้เคียงกัน เช่น พื้นที่เกษตรบางช่วงมีลักษณะคล้ายป่า หรือพื้นที่ก่อสร้างมีลักษณะคล้ายดินเปล่า
### 4.2 พื้นที่ที่โมเดลมีความไม่แน่นอน
พื้นที่ที่โมเดลมีความไม่แน่นอนสูง ได้แก่
-	พื้นที่รอยต่อระหว่างเมืองและเกษตร
-	พื้นที่ชานเมือง
-	พื้นที่เกษตรที่มีน้ำผสม
### 4.3 การประเมินความน่าเชื่อถือของแผนที่
จากผลการทดลอง
-	Accuracy ≈ 78–79%
-	Kappa ≈ 0.72–0.73
สามารถสรุปได้ว่าแผนที่มีความน่าเชื่อถือในระดับหนึ่ง เหมาะสำหรับการใช้งานเชิงภาพรวม (general analysis) แต่ควรใช้ด้วยความระมัดระวังในงานที่ต้องการความแม่นยำสูง
# ตอบคำถาม
### 1. ถ้าเพิ่ม Training Samples อีก 2 เท่า ความแม่นยำจะเพิ่มขึ้นเท่าไหร่? ทดสอบและอธิบายผล
การเพิ่ม Training Samples ช่วยให้โมเดลเรียนรู้ได้ดีขึ้น ทำให้ Accuracy มีแนวโน้มเพิ่มขึ้น แต่จากลักษณะข้อมูลที่มีความซ้ำกันในเชิงพื้นที่ คาดว่าจะเพิ่มขึ้นเพียงเล็กน้อย (ประมาณ 1–5%) หากข้อมูลใหม่ไม่หลากหลาย
### 2. Spatial Autocorrelation มีผลต่อความแม่นยำอย่างไร?
Spatial Autocorrelation อาจทำให้ค่าความแม่นยำสูงเกินจริง เนื่องจากข้อมูล Train และ Test มีความใกล้เคียงกันในเชิงพื้นที่ วิธีที่เหมาะสมในการแก้ปัญหานี้คือการใช้ Spatial Cross-validation
### 3. Class ใดที่โมเดลทำได้แย่ที่สุด — แก้อย่างไร?
จากค่า F1-score พบว่า Class ที่มีประสิทธิภาพต่ำคือกลุ่ม Barren หรือบาง Class ที่มีตัวอย่างน้อย
แนวทางแก้ไข ได้แก่
-	เพิ่มจำนวน Training Samples
-	ปรับปรุงการกำหนด Class ให้ชัดขึ้น
-	เพิ่ม Feature เช่น texture หรือ temporal data
### 4. ถ้าทำพื้นที่อื่น ต้องเปลี่ยนอะไร?
ในการนำวิธีการนี้ไปใช้ในพื้นที่อื่น
สิ่งที่ต้องปรับเปลี่ยน:
-	ขอบเขตพื้นที่ (ROI)
-	Training Samples
สิ่งที่สามารถใช้ซ้ำได้:
-	โครงสร้างโค้ด
-	ขั้นตอน preprocessing
-	วิธีการสร้าง Feature และการฝึกโมเดล
